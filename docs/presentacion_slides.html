Contexto de negocio

El problema: campañas que se deciden sin datos

  • Un banco ejecuta campañas telefónicas para captar depósitos a plazo, pero no sabe qué perfiles ni qué canales convierten.
  • Objetivo: transformar datos crudos de campañas en información accionable para tres audiencias distintas — gerencia, analistas y personal operativo.
  • Punto de partida real: 4.521 contactos con solo 11,5% de conversión → hay margen claro para focalizar el esfuerzo comercial.
Valor: pasar de “llamar a todos” a “llamar a quien más probabilidad de conversión tiene”, con evidencia y no por intuición.

Bank Marketing Analytics · SCY110104
Pipeline ETL · Indicador clave

Robustez: integra, valida y tolera fallos

  • 3 fuentes reales orquestadas: CSV, API REST del World Bank (Portugal, 2018–2024) y tabla SQL de referencia.
  • Validación de esquema: columnas requeridas, tipos, dominios binarios, nulos y outliers.
  • Trazabilidad: cada corrida se registra en etl_runs con estado y métricas de calidad.

Manejo avanzado de errores

  • Reintentos con backoff exponencial + timeout en la API externa
  • Fallback a caché local si la red falla
  • Transacciones con rollback en la carga
  • Logging profesional con rotación de archivos
